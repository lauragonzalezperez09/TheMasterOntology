Abstract

The ontology was developed using the Linked Open Terms (LOT) methodology, a lightweight approach for creating and publishing ontologies for the Semantic Web. During the specification phase, an Ontology Requirements Specification Document (ORSD) was created to define the purpose, scope, and requirements. Conceptualization was carried out using draw.io with the Chowlk plugin, reusing classes and properties from existing ontologies such as FOAF, European Learning Model Ontology, and FRAPO. Implementation was performed in Protégé, utilizing Chowlk Converter and OpenRefine for data generation and transformation. Finally, publication was done with Ontoology, allowing for the documentation and dissemination of the ontology, making it ready for maintenance and evolution.
